# 📝 就活ES作成フレームワーク

Cursor AI と連携して、効率的にES（エントリーシート）を作成するためのフレームワークです。

> ⚠️ **前提条件**: このフレームワークは [Cursor](https://cursor.sh/) エディタの使用を前提としています。Cursor の AI 機能と `.cursor/rules` を活用することで、最大限の効果を発揮します。

## 🎯 このフレームワークの特徴

1. **構造化された自己分析** - 何を書けばいいかが明確
2. **AI連携** - `.cursor/rules` でCursorがES作成をサポート
3. **エピソードの再利用** - 一度書けば複数企業に使い回せる
4. **進捗管理** - 選考状況が一目でわかる
5. **一貫性のあるES生成** - `.cursor/rules` により、AIが読み込むべきファイル（人物像・エピソード・自己分析など）を自動で把握し、どの企業のESを書いても一貫した内容・トーンで作成できる

## 🚀 使い方

### Step 1: 人物像まとめを記入

`人物像まとめ.md` を開き、以下を記入してください：

- 基本情報（名前、所属、スキルなど）
- 大事にしていること
- 強みと弱み
- 仕事選びの軸
- 将来像

### Step 2: エピソードを整理

`自分のこと/ESテンプレート/ポートフォリオ/` に、アピールできるエピソードを記入：

- 開発経験（個人開発、チーム開発）
- アルバイト経験
- サークル・部活動
- 学業での取り組み

### Step 3: 自己分析を深める

`自分のこと/ESテンプレート/自己分析/` に、以下を記入：

- 長所・短所
- 挫折経験
- 就活の軸

### Step 4: 企業ごとのESを作成

1. `企業/選考中/[企業名]/` フォルダを作成
2. Cursorに「〇〇企業のESを書いて」と依頼
3. AIが適切なエピソードを選んでESを生成

## 📋 ESの構成テンプレート

### 自己PR
1. 結論（私の強みは〇〇です）
2. その強みを発揮した具体的なエピソード
3. どのような行動・工夫をしたか
4. その結果どうなったか
5. 入社後どう活かすか（あれば）

### ガクチカ（学生時代に力を入れたこと）
1. 何に取り組んだか（結論）
2. なぜそれに取り組んだか
3. 具体的に何をしたか（行動・工夫）
4. 結果どうなったか
5. そこから何を学んだか

### 志望動機
1. 志望理由（結論）
2. なぜその業界か
3. なぜその企業か（企業の特徴に触れる）
4. 入社後にやりたいこと

---

## 📏 文字数調整のルール

| 文字数 | 構成 |
|-------|-----|
| 100字以下 | 結論のみ |
| 200字 | 結論 + エピソード概要 |
| 400字 | 結論 + エピソード + 工夫・学び |
| 600字以上 | 結論 + エピソード + 工夫 + 結果 + 今後への活かし方 |

---

## ✍️ 文体・表現のルール

1. **一人称は「私」に統一**する
2. **です・ます調**に統一する
3. **結論を先に書き**、そのあとに理由と具体的なエピソードを書く
4. 抽象的な表現だけで終わらせず、**行動内容・工夫した点・結果**まで書く
5. **専門用語（論理的思考力、数学力、コミュニケーション能力など）は使わない**
6. 強み・経験を盛るときも、**事実に基づいた範囲**で誇張しすぎない
7. 質問文を踏まえて、**聞かれていないことを長く書きすぎない**
8. 企業ごとの志望理由は、**その企業の特徴に触れた内容**にする
9. **一貫性を持つ**こと（同じ質問には同じ軸で答える）
10. ESの中で同じことを繰り返さない

##  Cursor AIとの連携

`.cursor/rules` にES作成ルールが設定されています。

### 使い方の例

```
「サイバーエージェントの自己PR（400字）を書いて」
「ガクチカを技術系エピソードで書いて」
「志望動機を300字で書いて」
```

AIは以下を自動で行います：
- 適切なエピソードの選択
- 企業タイプに応じた強調ポイントの調整
- 文字数に応じた構成の調整

## 📂 ファイルの記入順序

1. `人物像まとめ.md` ← **最初にここから**
2. `自分のこと/ESテンプレート/ポートフォリオ/` 内のエピソード
3. `自分のこと/ESテンプレート/自己分析/` 内の各ファイル
4. `自分のこと/ESテンプレート/エピソード対応表.md`
5. 企業ごとのES作成

---

## 💡 Tips

- **同じESで同じエピソードを2回使わない**（切り口を変える）
- **数値は正確に記載**（期間、人数など）
- **企業の特性に合わせて強調ポイントを変える**
  - 技術系企業 → 設計の話、技術選定の理由
  - コンサル系 → チームワークの話、課題解決プロセス
  - ベンチャー → スピード感、やり切る力
